A simple matching engine for limit orders

(Link)

In August 2025, I started seriously aiming for quantitative finance roles, and so decided to create what is probably the most common quant dev project.

This is a price-time priority matching engine written in C++. I implemented basic functionality for Good-Till-Cancel limit orders. Unit testing is done with Google test.

The design of this order book is largely influenced by this article from 2011, especially the use of ordered maps for storing limits, and a lock-free ring buffer for updating external observers.

It's quite a small project, but it took me a good amount of effort due to my relative lack of experience with C++ at the time. This is the project which best represents my C++ skills.

Potential future to-dos:

  • Add more order types (Fill-Or-Kill, Market Orders, Stop Orders)
  • Create a more thorough API for getting order data.
  • Test its performance much more thoroughly using realistic sample data and better profiling tools.
